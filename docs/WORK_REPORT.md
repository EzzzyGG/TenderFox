Дата: 2026-03-14

## 1) Коротко: где мы сейчас

На сегодня TenderFox — это **живой MVP-каркас**, который включает:

- запущенную связку **Docker Compose** (API + Postgres + Redis)
- **миграции Alembic** и рабочую структуру БД
- рабочие **API эндпоинты** (в т.ч. `/subscriptions`, `/search`, `/tenders/...`)
- тестовый источник (GosPlan v2) и базовую нормализацию

## Обновление статуса (2026-03-19)

### Проделанная работа (последние PR)
- Миграция подписок на `user_id` + JWT-only доступ к `/subscriptions`.
- Scheduler переведён на доставку по `telegram_links` (теперь `chat_id` берётся из `telegram_links` по `user_id`).
- Telegram bot: добавлен флоу привязки `/verify`: ввод телефона → отправка контакта → `POST /auth/verify_telegram_contact`.
- БД: добавлена таблица `telegram_pending_phones` для временного хранения введённого телефона (TTL 10 минут).
- Данные/схема: `subscriptions.user_id` сделан `NOT NULL`.
- Legacy удалён: колонка `subscriptions.chat_id` удалена, legacy fallback в scheduler убран.

### Текущий статус готовности
Оценка готовности **MVP end-to-end** (поиск → фильтрация → подписка → доставка в Telegram): **~70%**.

Что уже работает:
- API (FastAPI) + Postgres + Alembic миграции
- Базовый поиск/получение тендеров и сохранение в БД
- JWT auth + привязка Telegram через контакт
- Scheduler отправляет уведомления в Telegram по активным подпискам

Что ещё не доведено до “полноценного сервиса”:
- качество поиска/нормализация/ранжирование
- устойчивость источников и планировщика (ретраи, таймауты, лимиты)
- UI/кабинет (создание/управление подписками без Swagger)
- наблюдаемость (метрики/алерты) и эксплуатация

### Следующие действия (чтобы получить полноценный рабочий сервис)
1) **Стабильный ingestion**
   - добавить ретраи/бэкофф/таймауты для источника (httpx) и логировать ошибки по категориям
   - хранить `last_seen`/watermark по источнику, чтобы не гонять одно и то же
   - покрыть тестами `normalize_purchase` + контракт на поля (title/price/deadline/url)

2) **Поиск и фильтры**
   - ввести нормальные фильтры подписок: ключевые слова, исключения, диапазоны цен, даты, регионы
   - индексы в Postgres под поиск (GIN/trgm по `tenders.title`, индексы по `published_at`, `deadline_at`, `region`)
   - пагинация и лимиты в API

3) **Доставка и антидубли**
   - подтверждение, что `delivery` гарантирует dedupe (unique constraint по subscription_id+tender_id)
   - настройка частоты рассылки, rate-limit Telegram (429 retry)

4) **Продуктовый кабинет (без Swagger)**
   - минимальный UI (web/ или Next.js) для:
     - логина по телефону
     - создания/удаления подписок
     - просмотра последних доставок

5) **Прод-эксплуатация**
   - конфиг через env, секреты вне репо
   - мониторинг: логи, метрики (кол-во доставок, ошибки источника, 429 Telegram)
   - healthchecks для api/scheduler/bot, автоперезапуск
