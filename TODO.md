# Стратегический план развития (12 месяцев)

## Q1 — MVP и первые продажи (через таргет/рекламу)

### 1) Цель Q1
Сделать работающий агрегатор тендеров: поиск → фильтр → подписка → уведомления (Telegram).

### 2) MVP (обязательное)
- [ ] Источник данных: интеграция с zakupki.gov.ru (поиск + карточка тендера)
- [ ] Нормализация результата (единый JSON формат)
- [ ] БД Postgres: таблицы tenders / subscriptions / deliveries
- [ ] API:
  - [ ] `GET /search?keyword=&region=` — реальные результаты
  - [ ] `GET /tenders/{id}` — детали
  - [ ] `POST /subscriptions` — создать подписку
  - [ ] `GET /subscriptions` — список подписок
- [ ] Telegram:
  - [ ] бот + onboarding (получить chat_id)
  - [ ] отправка уведомлений по подписке
- [ ] Планировщик:
  - [ ] Celery worker + beat (или cron) для периодической проверки
  - [ ] защита от дублей (dedupe по tender_id + subscription_id)
- [ ] Запуск:
  - [ ] Dockerfile + docker-compose (api + postgres + redis + worker)
  - [ ] `.env.example`

### 3) “Пипл-френдли” требования (в MVP)
- [ ] Понятная выдача: короткий заголовок + цена + дедлайн + ссылка
- [ ] Чёткие фильтры без бюрократии: «ключевые слова», «регион», «минимальная цена»
- [ ] Простой сценарий: *нашёл → подписался → получаешь новые*
