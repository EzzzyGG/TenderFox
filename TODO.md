# Стратегический план развития (12 месяцев)

## Q1 — MVP и первые продажи (через таргет/рекламу)

### 1) Цель Q1
Сделать работающий агрегатор тендеров: поиск → фильтр → подписка → уведомления (Telegram) + сайт-лендинг.

### 2) MVP (обязательное)
- [x] Источник данных: рабочий REST-источник (через GosPlan v2 test как обёртку над ЕИС/zakupki)
- [x] Нормализация результата (единый JSON формат)
- [x] БД Postgres: таблицы tenders / subscriptions / deliveries
- [x] Alembic миграции
- [x] API:
  - [x] `GET /search?keyword=&region=` — реальные результаты
  - [x] `GET /tenders/{id}` — детали (кеш в БД)
  - [x] `POST /subscriptions` — создать подписку
  - [x] `GET /subscriptions` — список подписок
- [ ] Telegram:
  - [ ] бот + onboarding (получить chat_id)
  - [ ] отправка уведомлений по подписке
- [ ] Планировщик:
  - [ ] Celery worker + beat (или cron) для периодической проверки
  - [ ] защита от дублей (dedupe по tender_id + subscription_id)
- [x] Запуск:
  - [x] Dockerfile + docker-compose (api + postgres + redis)
  - [x] `.env.example`

### 3) “Пипл-френдли” требования (в MVP)
- [x] Понятная выдача: короткий заголовок + цена + дедлайн + ссылка
- [x] Чёткие фильтры без бюрократии: «ключевые слова», «регион (код)», «минимальная цена»
- [ ] Сайт-лендинг: один оффер + CTA «Открыть в Telegram»
- [ ] Мини-кабинет: показать подписки
